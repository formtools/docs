{% include screenshot.html item="install_step2.png" %} The second step runs a few key tests on your server to confirm that Form Tools is able to run. The full system requirements are outlined here, but the most common problems encountered on this page are these:

  • MySQL version identified as incorrect. Form Tools requires MySQL 4 or later to run.
  • Write permissions. The /themes/default/cache and /upload folders needs to be read-writable. If you're on a Windows server, you may need to contact your hosting provider to ask them to set the folder permissions.

The last row lets you choose a custom cache folder if you don't want to use the default /cache folder. This folder is used for storing temporary cached version of the Form Tools interface pages amongst other things. It is perfectly safe to have it within your webroot, but you can put it wherever you need.
