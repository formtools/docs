The Core Field Types

The Form Tools Core come pre-bundled with the field types listed below. Between them, these field types cover all of the most typical fields needed in people's forms. Each field type comes prefigured with the appropriate settings and validation so no additional configuration is needed.

  • Textboxes
  • Password fields
  • Textareas
  • Radio Buttons
  • Checkboxes
  • Dropdown fields
  • Multi-select dropdowns
  • Dates
  • Times
  • Phone Number fields
  • Code / Markup fields

This module lets you edit any of these Core field types, with the exception of textboxes and dates. Those two field types have hardcoded references in the Core (for the Submission ID, Last Modified, Submission Date and IP Address) and cannot be changed. Instead, the module allows you to make copies of them, which you can then edit.

Additional Field Types

In addition, field types may also be created via separate modules. This has several benefits over using the Custom Fields module, but it's considerably more complex. The main Form Tools download package, contains two additional field type modules:

  • File Upload field
  • TinyMCE (WYSIWYG) field

Any module that extends the Core with their own field types cannot be edited via the Custom Fields module. You can use the module to browse their configurations to see how they've been set up, but you won't be able to update them. This ensures that you never make changes to those field type configurations which are later overwritten by updates to the module.
